### Key Decisions

1. **Ownership model**: Field-level ownership (not entity-level)
   - GitHub owns: title, description, assignees
   - Forge Pipeline owns: status, dueDate, riskState
   - Bidirectional: priority, tags/labels

2. **Conflict resolution**: 
   - Check ownership rules
   - Check last-modified timestamp
   - GitHub wins for content
   - Manual queue for unresolved conflicts

3. **State storage**: SQLite + Redis for sync state and conflict tracking

4. **Sync approach**: Webhook-driven, not polling
