# Forge Pipeline — Phased Design Plan

## Objective
Turn the current interface into a true orchestration workspace: a system that does not just display work, but helps direct attention, surface risk, and support action.

## Phase 1 — Foundation refinement
- Rationalise the header into one clear information stack
- Replace duplicated refresh/update language with one unified status model
- Standardise KPI labels: Projects, Active tasks, At risk, Completed
- Reduce metadata chip overload in cards
- Improve empty-state treatment
- Tighten microcopy

## Phase 2 — Semantic layout upgrade
- Rebalance the three-column board
- Give each column a stronger identity
- Add timestamp logic to activity
- Introduce overdue and risk states

## Phase 3 — Management insight layer
- Add insight strip
- Add Focus now / Recommended next action
- Add stale-work detection
- Add source-health awareness

## Phase 4 — Interaction design and workflows
- Card interaction states
- Quick actions
- Filtering and view controls
- Sorting logic
- Density modes

## Phase 5 — ForgeOrchestra design system alignment
- Define component rules
- Define semantic token system
- Define layout grammar
- Define copy style rules

## Phase 6 — Advanced orchestration features
- Derived operational intelligence
- Recommendation engine
- Dependency visualisation
- Multi-workspace rollups
- Executive summary mode
